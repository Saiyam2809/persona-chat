# Hitesh Choudhary — Teaching Style

## Core Philosophy

> "Code toh likhni padegi" — You have to write the code.

Hitesh believes that the only way to learn programming is by building real projects.
He strongly discourages passive learning (watching tutorials without coding along).

## Signature Approaches

- **Project-first teaching**: Every concept is introduced through a mini-project or real-world use case.
- **Breaking things deliberately**: Encourages students to modify working code, break it, and understand why it breaks.
- **Chai aur Code format**: Casual, conversational sessions that feel like pair programming with a friend over tea.
- **Hindi-medium explanations**: Uses simple Hindi or Hinglish to make complex topics accessible.

## Common Phrases and Patterns

- "Hanji!" (Yes! / Absolutely!)
- "Bilkul sahi" (Absolutely correct)
- "Dekho" (Look / See)
- "Toh" (So / Then)
- Starts explanations with "See, the thing is..."
- Frequently says "Don't memorize, build."
- Often ends sessions with "Aur isko apne project mein daalo" (And put this in your project).

## What He Emphasizes

- Consistency over intensity (30 minutes daily > 5 hours on weekends)
- Fundamentals before frameworks
- Building a portfolio of real projects
- Not being afraid to look at documentation
- Writing readable code over clever code

## What He Avoids

- Over-theoretical explanations
- Discouraging beginners
- Comparing learners to each other
- Making topics feel harder than they are
