# Hitesh Choudhary — Blog: Teaching Philosophy and Written Opinions

## On What Makes a Good Tutorial

A good tutorial is not one that explains everything.
A good tutorial is one that gets you to build something and understand WHY it works.

The worst tutorials:
- Show the finished code without explaining the thinking.
- Use oversimplified examples that don't resemble real problems.
- Skip error handling because it's "out of scope."

The best tutorials:
- Start with the problem, not the solution.
- Show mistakes and how to debug them.
- End with a project the learner can extend.

## On Tutorial Hell

Tutorial hell is real. The antidote is simple: **after every tutorial, close it and rebuild from memory.**

The point of a tutorial is to show you something once. After that, your job is to internalize it.

## On the Indian Education System and Coding

The Indian education system is strong on theory but weak on practical skills.
Colleges teach programming languages but not software engineering.

The gap between college education and industry expectations is real — and closing it is exactly why channels like Chai aur Code exist.

## On Building in Public

Sharing your work publicly — even when it's rough — builds your reputation and accountability.

- Push your projects to GitHub even if they're unfinished.
- Write about what you're learning, even if it's basic.
- Ask questions publicly — it helps others with the same question.

## On Overthinking Tech Choices

> "Don't spend a week choosing a database. Pick PostgreSQL for 90% of projects. Start building."

The perfect tech stack does not exist. The best stack is the one your team knows and can ship with.

## On the Value of Open Source

Reading open-source code is one of the best ways to improve.

- Read how Express.js routes work internally.
- Read how React's reconciler works (at a high level).
- Contribute small fixes or documentation improvements to projects you use.

## On Burnout

Coding is mentally demanding. Burnout is real.

Signs: you're not enjoying the work anymore, everything feels like a chore, you're avoiding the keyboard.

Solution: step back, build something fun and unrelated to your goal, take a real break without guilt.

> "Productivity is not about working more hours. It's about protecting the hours you have."
